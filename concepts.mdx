## Platform Architecture

Bastion is built on a modular architecture that allows organizations to adopt only the security capabilities they need while maintaining a unified view of their security posture.

## Workspaces

A **Workspace** represents a single organization or customer environment in Bastion. Each workspace contains:

- Employees and compliance groups
- Integrations and connected services
- Compliance frameworks and evidence
- Security configurations and policies
- Device inventory and MDM settings


## Modules Overview

Bastion's functionality is organized into distinct modules:

    - **People** - Employee directory synced from identity providers

    - **Phishing Campaigns** - Simulated phishing exercises

    - **Typosquatting** - Domain squatting detection

## User Roles

Bastion uses role-based access control (RBAC):

| Role | Description | Access Level |
|------|-------------|--------------|
| **Admin** | Full platform access | All modules and settings |
| **Compliance Manager** | Compliance-focused access | Compliance, policies, audits |
| **Security Analyst** | Security monitoring | Infrastructure, devices, monitoring |
| **Employee** | Limited access | Training, assigned tasks |

## Data Model

### Compliance Groups

Groups organize employees for:
- Training assignment
- Phishing campaign targeting
- Access review scoping
- Policy distribution

### Evidence

Compliance evidence is:
- Automatically collected from integrations
- Manually uploaded for custom controls
- Versioned and timestamped
- Linked to specific framework controls

### Policies

Policies flow through a lifecycle:
1. **Draft** - Initial creation
2. **Review** - Stakeholder review
3. **Approved** - Ready for distribution
4. **Distributed** - Sent to employees
5. **Acknowledged** - Employee confirmation
