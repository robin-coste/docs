## Overview

ITSM integrations connect Bastion to your ticketing systems. Automatically create tickets for security issues, track remediation, and maintain audit trails.

## Jira

### Prerequisites

- Jira Cloud or Server access
- Project admin permissions
- API token (Cloud) or credentials (Server)

### Setup Steps

<Steps>
  <Step title="Generate API Token">
    Create API token in Atlassian account settings.
  </Step>
  <Step title="Connect in Bastion">
    Enter Jira URL, email, and API token.
  </Step>
  <Step title="Select Project">
    Choose target project for tickets.
  </Step>
  <Step title="Map Fields">
    Configure field mappings.
  </Step>
  <Step title="Test">
    Create a test ticket.
  </Step>
</Steps>

### Configuration

| Setting | Description |
|---------|-------------|
| **Project** | Target Jira project |
| **Issue Type** | Bug, Task, Security Issue |
| **Priority Mapping** | Severity to priority |
| **Assignee** | Default or dynamic |
| **Labels** | Auto-applied labels |

### Field Mapping

Map Bastion fields to Jira:

| Bastion Field | Jira Field |
|---------------|------------|
| Title | Summary |
| Description | Description |
| Severity | Priority |
| Asset | Custom field |
| CVE ID | Custom field |

## Auto-Ticket Creation

### Trigger Events

Automatically create tickets for:

| Event | Ticket Type |
|-------|-------------|
| **New Critical Issue** | Security incident |
| **Vulnerability SLA Breach** | Task |
| **Access Review Finding** | Change request |
| **Compliance Gap** | Task |

### Trigger Configuration

1. Go to **Settings** → **ITSM Triggers**
2. Click **Add Trigger**
3. Select event type
4. Configure conditions
5. Map to ticket type
6. Enable

### Ticket Content

Tickets include:

- Issue title and description
- Severity and priority
- Affected asset
- Remediation guidance
- Link back to Bastion

## Status Sync

### Bi-Directional Sync

| ITSM Action | Bastion Update |
|-------------|----------------|
| Ticket resolved | Issue marked fixed |
| Ticket closed | Issue closed |
| Ticket reopened | Issue reopened |

| Bastion Action | ITSM Update |
|----------------|-------------|
| Issue fixed | Update ticket |
| Issue closed | Close ticket |
| Comment added | Add comment |

### Status Mapping

Map statuses between systems:

| Bastion Status | Jira Status | ServiceNow State |
|----------------|-------------|------------------|
| Open | To Do | New |
| In Progress | In Progress | Work in Progress |
| Fixed | Done | Resolved |
| Closed | Done | Closed |

## Workflow Integration

### Remediation Flow

```
Issue Found → Ticket Created → Assigned → Fixed → Verified → Closed
```

### SLA Tracking

Track against SLAs:

- Time to assignment
- Time to resolution
- Age warnings
- Escalation triggers

### Ticket Metrics

- Status changes
- Comments and updates
